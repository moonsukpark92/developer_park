01

Chapter One

나는 코딩을 한 줄도
모른다. 그런데 시작했다.

2026년 2월. 나는 친환경 투수보도블록을 만드는 회사의 대표다. 공장은 단양에, 본사는 부산에 있다. 직원들의 업무일지는 말로 처리되고, 견적서는 Excel을 이메일로 주고받는다. 현장관리는 전화로 한다.

어느 날 Claude라는 AI를 써봤다. "파이썬으로 파일 정리 프로그램 만들어줘"라고 입력했더니, 진짜 코드가 나왔다. 그리고 그 코드는 실제로 작동했다.

그 순간부터 나의 한 달이 시작됐다. 코딩을 배우러 간 게 아니었다. 코딩 없이 개발자가 되려고 했다. 이게 '바이버 코딩'이다.

음성으로 녹음하고, 텍스트로 붙여넣고, Claude가 처리한다. 이동 중에도, 골프장에서도, 차 안에서도. 생각하는 모든 순간이 개발 시간이 됐다. 나중에 이걸 이렇게 표현했다. "개떡같이 말해도 찰떡같이 알아듣는다."

바이버 코딩이란?

Vibe Coding — 코드의 원리를 이해하지 않고, 의도와 목적을 AI에게 말로 설명해서 원하는 프로그램을 만드는 방식. 도구는 Claude, Cursor, Windsurf. 배경지식은 0. 결과물은 실제로 작동하는 프로그램.

02

Chapter Two

에러와의 전쟁.
두려움이 선생님이었다.

순탄하지 않았다. 처음 2주는 에러의 연속이었다. 그런데 신기한 일이 일어났다. 에러가 날 때마다 Claude에게 스크린샷을 붙여넣었다. 그리고 물었다. "이게 무슨 소리에요?"

대부분은 10분 안에 해결됐다. 에러는 문제가 아니었다. 에러는 대화의 시작이었다.

Error #001 — Cowork

RPC error -1: failed to ensure virtiofs mount: Plan9 mount failed

Cowork가 폴더 인식 불가. 홈 디렉토리 이동 + 앱 재시작으로 해결. 총 4번 반복.

Error #002 — Claude Code

command 'claude-vscode.editor.openLast' not found

VS Code 확장 설치 후 재시작 누락. 재설치 + API 키 재설정.

Error #003 — Python 3.9

TypeError: X | None not supported (서버 배포 시)

가비아 서버가 Python 3.9. X|None → Optional[X]로 전체 수정.

Error #004 — HTTPS

Mixed Content blocked — http API on https site

.env 파일 2개 수정. CORS_ORIGINS + REACT_APP_API_URL 모두 https로.

Error #005 — Tailwind CSS

CSS 파일 306B (정상: 5.32KB) — 스타일 전체 깨짐

config 파일 누락. 임시 디렉터리에서 빌드 후 복사로 해결.

Error #006 — OAuth

Google OAuth Error 400: redirect_uri_mismatch

Cloud Console URI와 실제 URI 불일치(http↔https, 포트, 슬래시). 확인 후 해결.

에러가 쌓일수록 이상하게 두려움이 사라졌다. 에러는 내가 뭔가를 시도했다는 증거였다. 에러가 없는 날은 아무것도 시도하지 않은 날이었다.

03

Chapter Three

한 달 동안
실제로 만든 것들

에러와 싸우면서도 회사에서 쓰는 프로그램들이 하나씩 만들어지기 시작했다. 코딩을 모르는 대표가 설계하고, Claude가 구현했다. 전부 실제로 쓰고 있다.

🗂 FileClassifier V5.0 — 정확도 89.4%

596건 데이터로 학습. 규칙+ML 하이브리드. 77% 자동화. 생산본부/출하팀 100% 정확도. 1일 개발, 비용 0원.

📊 매출 일일마감 보고서 V4.11

ERP 출하요청 → 40컬럼 정규화 → 5시트 보고서 자동 생성. Python Flet. 매일 실제 사용 중.

🏭 생산출하 분석 시스템

ERP/MES 4종 파일 자동 분류·병합·분석. 2024~2026년 데이터 통합. BarChart/LineChart 대시보드.

📱 월자금계획 자동화

자금일보+수금계획+급여대장+법인카드+매입품의 통합 → 월간 자금계획서 자동 생성. EXE 배포 완료.

🍽 사내식당 식단표 자동화

레시피 143개, 식자재 135개 DB. 주간 식단 입력 → 발주 자동 계산 → 엑셀 출력. 외부 도시락 집계까지.

🗑 유틸리티 7종 EXE 배포

DuplicateCleaner, EmptyFolderCleaner, FileYearOrganizer, FolderMapper, FileRenamer 등. 전사 배포 완료.

핵심 발견

코딩을 배운 게 아니다. AI에게 원하는 것을 설명하는 법을 배운 거다. 설명이 정확할수록 결과물이 정확해진다. 이게 AI 시대의 기술이다.

Chapter Four

가장 중요한 것 하나.
CEO가 직접 해야 한다.

주변에서 묻는다. "왜 직원한테 안 시켜요?" 대답은 간단하다. 직원은 AI가 자기 일자리를 위협한다고 느낀다. 의식적이든 무의식적이든. 그래서 제대로 쓰이지 않는다.

CEO가 먼저 써야 한다. CEO가 먼저 익숙해져야 한다. CEO가 먼저 시스템을 만들어야 한다. 그래야 조직이 따라온다.

한 가지 더. AI 기술은 주 단위로 바뀐다. 지금 배우지 않으면 2주 후엔 이미 뒤처진다. 지금 하고 있는 사람과 나중에 하려는 사람의 격차는 매주 벌어지고 있다.
