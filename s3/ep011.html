Chapter 01

첫 번째 시도 — 그리고 실패

아이디어는 단순했다. 데코허브는 손댈 게 계속 나오는데, 시간도 없고 에러도 자꾸 챙겨야 되고. 그러면 이걸 스스로 하는 에이전트를 만들자.

울트라9 노트북에 GPU가 있으니까, 로컬 LLM을 올려서 해보자. 코딩도 LLM, 검수도 LLM, 2차 검수만 Claude. 완벽해 보였다.

[00:00] LLM 서버 구동 시작 — GPU 할당 중... [00:03] 배터리 잔량: 87% → 급격 감소 감지 [01:20] 배터리 10% 미만 — 전원 연결 중에도 공급 초과 [06:40] 밤새 실행 완료 [06:41] 결과물 검토 — 품질 기준 미달. 전면 폐기.

데이터센터가 에너지 귀신이라는 말, 직접 경험했다. 전원을 꽂아 놨는데도 배터리가 10%를 넘지 않았다. 그리고 밤새 돌아간 결과는 — 쓰레기였다. 로컬 LLM은 폐기했다.

Chapter 03

새벽 5시 — "너부터 업그레이드해"

자고 일어났다. 새벽 5시. 문득 드는 생각.

데코허브 에이전트, 얘는 자기 자신을 업그레이드 못 시키나? 한 시간 반 동안 얘랑 대화했다. "데코허브 시스템 업그레이드하기 전에, 니부터 업그레이드해."

메모리 구조, 자기 학습 방법, 자기 진단 로직. 요즘 핫한 Prompt Engineering, LLM Wiki 3계층 구조, Memory 아키텍처까지 — 최신 기술로 자기 자신을 고치기 시작했다.

Chapter 04

기억을 설계하다

세션이 끊기면 맥락이 사라진다. 토큰이 차면 압축된다. 이 '쓸데없는 이야기들'이 사실은 쓸데없는 게 아니었다.

AI와의 대화 속 잡담들 — 그게 맥락을 이해하는 가장 좋은 키워드다. 내가 어떤 생각을 가진 사람인지, 어떻게 결정하는지. 그걸 AI에게 알려줘야 AI가 그 지침대로 움직인다.

못 받으면? 자기 판단으로 가장 객관적인 방향을 간다. 그게 내 의견과 달라진다.

[MEMORY] MD 파일 → 3계층 Wiki 구조 구성 중 [MEMORY] GitHub 링크 연동 → 프로젝트 컨텍스트 주입 [MEMORY] DECOHUB 에이전트 구조 → 문실장(문돌) 연계 설정 [MEMORY] CEO 사고방식 → 에이전트 철학 레이어에 반영 완료

링크만 주면 된다. 웬만한 건 다 링크 주고 던지면 알아서 알아먹는다. 그게 지금 이 시대다.

Chapter 05

같은 건 없었다

에이전트에게 30분을 줬다. 서브에이전트로 탐색해라. 내가 생각하는 자율 코딩 에이전트, 비슷한 게 있냐고.

같은 건 없단다.

참고할 스택은 5개 프로젝트 정도. 대형 프로젝트들에서 비슷한 시도가 있긴 한데 — 우리가 생각하는 아이디어의 핵심은 없다. 아마 불가능하다고 생각했겠지. 그래서 안 했을 수도.

모든 권한을 줬다. 야간에 시스템 최적화도 네가 알아서 해라. 지금 진행 중이다.

나는 코드를 하나도 모른다.
코드 모르는 사람이 굳이 앉아서 엔터 키 눌러주고 승인해줄 필요가 있나?

봐도 모르는데.

대신 — AI가 일할 수 있는 조건을 만드는 것.
그게 내가 할 일이다.
