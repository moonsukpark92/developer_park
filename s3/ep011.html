Chapter 01

20시간 — v1에서 v11까지

처음 설계는 이랬다. Claude가 기획하고, Ollama에 올린 Qwen 14B가 코딩하고, DeepSeek R1이 검수하고, 마지막에 Claude가 최종 확인. 로컬 LLM 3대를 조합한, 완벽해 보이는 파이프라인.

[v1] Claude CLI — PATH를 못 찾음. 실행 불가. [v2~v3] JSON 파싱 실패 — stdin 파일 기반으로 전환 [v4] 기획 성공 — 12개 태스크 자동 분해 완료 [v4] 텔레그램 보고 크래시. Ollama TCP 끊김. [v5] Ollama 서버 불안정 — 메모리 위반 반복 [v6] 15시간 50분 연속 자율 실행. 99커밋. 170파일. [v6] 13개 태스크 완료 — 그러나 품질 기준 미달. 전면 폐기.

v6에서 밤새 돌아간 결과는 — 쓰레기였다. 99개 커밋, 170개 파일을 고쳤는데, Qwen 14B의 코드 품질이 너무 낮았다. 데이터센터가 에너지 귀신이라는 말, 직접 경험했다. 전원을 꽂아 놨는데도 배터리가 10%를 넘지 않았다. Ollama와 로컬 LLM은 폐기했다.

v1부터 v11까지 20시간. 그 사이에 발견한 치명적 버그가 4개.

[BUG-1] claude-agent-sdk 패키지가 존재하지 않음 → CLI subprocess로 교체 [BUG-2] Python 3.10+ 타입힌트 사용 → typing 모듈로 전환 [BUG-3] IndexError — 배열 범위 초과 → 검증 로직 추가 [BUG-4] state.json race condition → tempfile + rename 원자적 쓰기
reset
Chapter 02

원점 — 7단계 자율 파이프라인

다시 생각했다. Qwen과 DeepSeek를 버리고, 검증된 조합으로. Claude Opus가 기획과 코딩을 통합하고, OpenAI Codex(gpt-5.3-codex)가 1차 검수. 오케스트레이션은 LangGraph.

// DECO AGENT ARCHITECTURE v11 — LangGraph
NODE 01
Planner · Opus 4.6
계획서(.md) → TaskItem 리스트 자동 분해
→
NODE 02
Coder · Opus 4.6
멀티파일 자율 생성 + Git 자동 커밋
↓
NODE 03
Debugger
AST 구문 + import 검증, 자동 수정 (최대 5회)
→
NODE 04
Reviewer · Codex + Opus
듀얼 검수 + 크로스 보고서 (최대 3회 루프)
↓
NODE 05
Tester · Playwright
E2E 스크린샷 + 헬스체크 (/health)
→
NODE 06
Approval · Telegram
변경사항 + 스크린샷 → /approve 또는 /reject
↓
NODE 07
PR · GitHub
git push → gh pr create → PR URL 반환 → 365일 데몬

실패하면 이전 노드로 자동 복귀한다. 디버깅은 5회, 검수는 3회까지 자동 루프. LangGraph가 상태를 관리하고, 무한 루프를 방지한다. 조용히 움직이기 시작했다.

Chapter 03

새벽 5시 — "너부터 업그레이드해"

자고 일어났다. 새벽 5시. 문득 드는 생각.

데코허브 에이전트, 얘는 자기 자신을 업그레이드 못 시키나? 한 시간 반 동안 얘랑 대화했다. "데코허브 시스템 업그레이드하기 전에, 니부터 업그레이드해."

그래서 만든 게 3계층 Wiki 구조다. Karpathy의 LLM Wiki 패턴을 참고했다.

// 3-LAYER WIKI ARCHITECTURE

↓ 가공
LAYER 2

AI가 관리 — entities/(모듈별), concepts/(기술 결정), log.md(시간순)
↓ 학습
LAYER 3

PERSONA.md에 자기 정체성을 적고, BOSS_DIRECTIVES.md에 보스의 절대 규칙 21개를 기록하고, wiki/entities/에 아키텍처, 데몬, 인프라, 세션 이력을 정리했다.

Chapter 04

기억을 설계하다

세션이 끊기면 맥락이 사라진다. 토큰이 차면 압축된다. 이 '쓸데없는 이야기들'이 사실은 쓸데없는 게 아니었다.

AI와의 대화 속 잡담들 — 그게 맥락을 이해하는 가장 좋은 키워드다. 내가 어떤 생각을 가진 사람인지, 어떻게 결정하는지. 그걸 AI에게 알려줘야 AI가 그 지침대로 움직인다.

못 받으면? 자기 판단으로 가장 객관적인 방향을 간다. 그게 내 의견과 달라진다.

[WIKI] wiki/index.md — 전체 카탈로그 생성 [WIKI] wiki/entities/identity.md — 에이전트 정체성 기록 [WIKI] wiki/entities/architecture.md — 7노드 워크플로우 문서화 [WIKI] wiki/entities/infrastructure.md — 봇 군단 6대 인프라 기록 [WIKI] wiki/concepts/tech_decisions.md — Ollama 제거 사유 기록 [WIKI] wiki/concepts/failure_patterns.md — v1~v10 실패 패턴 분석 [MEMORY] memory/session_20260415_16.md — 20시간 세션 전체 기록 [MEMORY] memory/LEARNINGS.md — 자기 학습 인덱스 구축 완료

절대 규칙도 정했다. BOSS_DIRECTIVES.md에 21개. 1순위: 박문석 대표가 보스. 절대 협조. 2순위: 격리 환경 개발. 기존 코드 절대 수정 금지. 예외 없음. 이걸 어겼을 때가 보스가 가장 화났던 순간이었다.

Chapter 05

같은 건 없었다

에이전트에게 30분을 줬다. 서브에이전트로 탐색해라. 내가 생각하는 자율 코딩 에이전트, 비슷한 게 있냐고.

GitHub 15개 프로젝트, Anthropic 공식 문서, 커뮤니티 20건. Claude Agent SDK, Hooks, Agent Teams, Stripe의 Blueprint 패턴까지.

같은 건 없단다.

비슷한 시도는 있다. 대형 프로젝트들에서 부분적으로. 하지만 Opus 단일 LLM 기획+코딩, Codex+Opus 듀얼 검수, LangGraph 오케스트레이션, 3계층 Wiki 자기 학습, 365일 데몬, 텔레그램 자연어 소통 — 이 조합은 없다. 아마 불가능하다고 생각했겠지. 그래서 안 했을 수도.

모든 권한을 줬다. DEVPLAN_AGENT_V2.md에 17개 Phase를 적고, D-021 자동 승인 지시를 내렸다. 야간에 시스템 최적화도 네가 알아서 해라. 지금 진행 중이다.

나는 코드를 하나도 모른다.
코드 모르는 사람이 굳이 앉아서 엔터 키 눌러주고 승인해줄 필요가 있나?

봐도 모르는데.

대신 — AI가 일할 수 있는 조건을 만드는 것.
그게 내가 할 일이다.

모든 것을 기억하고 나눌 수 있는 존재를 만들고 싶었다.
