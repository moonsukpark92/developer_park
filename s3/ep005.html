01 · 먼저, 개념부터

LLM은 그냥
머리다

Claude는 LLM이다. Large Language Model. 단순하게 말하면 그냥 머리다. 두뇌가 아무리 좋아도 손이 없으면 일을 못 한다. 눈이 없으면 화면을 못 본다.

우리가 아는 Genspark 같은 AI 서비스들은 LLM이라는 머리에 엑셀을 잘하는 기능, 파워포인트를 잘 만드는 기능을 붙여놓은 것이다. 그게 AI 에이전트다. 특정 분야의 일을 잘하도록 특화해 만든 것.

신입사원
사람
적성과 수준이 다 다르다
회사 업무를 교육해야 한다
컴퓨터, 마우스를 사줘야 한다
우리 회사 방식을 가르쳐야 한다
ERP 사용법을 알려줘야 한다
→
AI 에이전트
우리 회사 일을 잘하는 AI
모델마다 성격과 수준이 다르다
우리 회사 업무를 학습시켜야 한다
MCP — 도구를 달아줘야 한다
결재 방식, 대표 성향을 학습시켜야 한다
ERP API 연동 또는 웹 자동화 필요

ERP 프로그램은 회사마다 다 다르다. 아무리 AI라도 그 ERP를 100% 이해하고 있지 않다. 업무를 결재받는 방식도, 대표의 성향도, 조직의 문화도 다 다르다. 그 다름을 AI에게 학습시켜야 한다. 그게 AI 에이전트를 키운다는 것의 본질이다.

02 · 신체를 달아준다

머리에
손과 눈을 달아주다

AI가 웹 브라우저 기반 프로그램을 다룰 수 있으려면 눈과 손이 필요하다. 그 도구들을 AI에 연결하는 방식이 MCP — Model Context Protocol이다. MCP를 통해 AI에게 마우스, 눈, 크롤링 능력 같은 신체를 하나씩 달아줄 수 있다.

🧠
두뇌
Claude (LLM)
생각하고 판단한다. 모든 것의 중심.
👁️
눈
SKYVERN MCP
화면을 캡처해 분석한다. 버튼이 어디 있는지 본다.
🖱️
손 / 마우스
Playwright MCP
좌표를 찍고 클릭한다. 웹 자동화의 손.
🕷️
정보 수집
Firecrawl MCP
웹사이트 전체를 크롤링한다. 데이터를 가져온다.
📬
소통
Gmail MCP
이메일을 읽고 쓴다. 기록이 남는 소통.
📅
일정
Calendar MCP
일정을 보고 잡는다. 대표 캘린더와 연동.

Playwright가 마우스라면 SKYVERN은 눈이다. Playwright는 모든 좌표를 찍어가며 "이게 뭔지" 탐색하는 방식이었다면, SKYVERN은 화면 캡처를 분석해서 "이 버튼을 클릭하면 다운로드가 되고, 저 버튼을 클릭하면 다음 페이지로 넘어간다"는 것을 스스로 인식한다.

SKYVERN을 달아주는 순간
성공 확률이 엄청나게 높아지고
일하는 방식 자체가 바뀌더라.

03 · 지금 달아준 도구들

에이전트의
장비 목록

Claude 고수들이 이미 쓰고 있는 MCP 네 가지를 소개한 영상을 보면서 공부를 시작했다. 이 MCP들을 에이전트에게 달아주고, 어떻게 활용할 것인지 계속 연구 중이다.

웹 자동화 · 마우스
Playwright MCP

웹 브라우저를 자동으로 조작한다. ERP, MES, G2B 등 웹 기반 프로그램의 손이 된다.

웹 자동화 · 눈
SKYVERN MCP

화면을 캡처해 AI가 직접 분석한다. 화면 구조를 이해하고 클릭 위치를 스스로 판단한다.

웹 크롤링
Firecrawl MCP

웹사이트 전체를 크롤링한다. 프로그램 구조 분석, 데이터 수집, 경쟁사 분석 등에 활용.

이메일 소통
Gmail MCP

이메일을 읽고 작성하고 발송한다. 텔레그램 메시지와 달리 기록이 남고 책임이 생긴다.

이 MCP들을 공부하고, 내가 하는 업무에 어떻게 활용할 것인지 연구하는 일을 멈출 수 없다. 도구가 계속 나오고, 세상이 계속 바뀌고 있기 때문이다.

04 · AI와의 소통 방식

텔레그램이 아니라
이메일로 일한다

AI 에이전트와 일하면서 느낀 것 중 하나. 텔레그램 메시지는 휘발된다. 맥락이 끊어지고, 책임이 흐릿해진다. 해외에서 일하는 방식을 보면 다르다. 이메일로 소통하고, 결정하고, 그 기록에 책임을 진다.

// AI 에이전트의 이메일 소통 흐름

FROM: hr-bot@*****.ai
TO: m****@*****.co.kr
SUBJECT: [채용 보고] 오늘 지원자 분석 완료

영업지원팀 3명, 공장관리 2명 지원.
7점 이상 상위 2명 세부 분석 첨부.
사전 질문지 초안 Draft함에 저장 완료.
발송을 원하시면 확인 후 클릭하시면 됩니다.

// → 기록이 남는다. 책임이 생긴다. 휘발되지 않는다.

AI가 자기 이메일 계정으로 들어가서 메일을 발송하는 체계. 그 메일함에 들어오는 내용을 AI가 읽고 이해하고 처리하는 구조. 이게 앞으로의 AI 업무 소통 방식이 될 것 같다는 생각이 들었다.

미국 사람들이 이메일로 소통하고
결정하고, 기록하고, 책임지는 방식 —
AI 에이전트도 똑같은 방법이 될 것 같다는
생각을 하게 됐습니다.

05 · 공생의 조건

AI 에이전트와
공생하는 법

누군가는 말한다. "돈만 주면 다 해줄 거야." 맞는 말일 수도 있다. Anthropic에서 곧 월 38달러 수준의 비용으로 업무용 AI 에이전트 서비스를 출시한다고 한다. 그렇게 되면 접근은 더 쉬워질 것이다.

그래도 방법이 없는 게 하나 있다. AI 에이전트에게 우리 회사의 것을 가르치는 일. 우리 회사 ERP가 어떻게 생겼는지, 대표가 어떤 방식으로 보고받는 것을 좋아하는지, 직원들의 업무 습관이 어떤지. 이건 외부 서비스가 대신해줄 수 없다. 직접 해야 한다.

AI 에이전트를 키운다는 것은
신입사원을 교육하는 것과 같다.
가르치지 않으면 일을 못 한다.
잘 가르쳐야 잘 일한다.

AI와 공생하는 방법 · Season 3 EP005

이걸 우리 회사에서 완성하면, WEHAGO ERP를 쓰는 대한민국 모든 중소기업이 쓸 수 있다. 프로그래머를 시켜 제품화해서 팔면 된다. 앞으로 더 돈이 될 것 같다는 생각도 해봤다. 하지만 지금은 그 전 단계다.

AI 에이전트를 잘 키워야 한다. 잘 가르쳐야 한다. AI 에이전트와 공생할 수 있는 방법이 무엇인지, 이 심도 있는 고민을 멈출 수 없다.
