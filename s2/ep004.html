Chapter 05

시야가 좁았다

Paperclip 하나에 흥분했다. 70% 일치. 소스코드 분석. 개발계획서 작성. 하지만 클로드가 말했다.

"맞습니다. 시야가 좁았습니다. 더 넓게 조사하겠습니다."

— Claude, 분석 확장 시작

클로드가 전체 생태계를 넓게 뒤졌다. 그리고 돌아온 답 —

내가 만들려는 건 하나의 프로그램이 아니었다.
5개의 서로 다른 기능 레이어가 결합된 시스템이었다. 각 레이어마다 참조할 최적의 프로그램이 달랐다.

레이어 핵심 기능 한줄 정의
① 에이전트 오케스트레이션 AI 에이전트 팀 조직화·위임·조율 "AI 직원들의 회사"
② 모니터링 & 옵저버빌리티 서버 상태, 에이전트 헬스, 이상감지 "AI 인프라 관제탑"
③ 프로젝트·태스크 관리 진행률, 담당자, 일정 추적 "AI 포함 프로젝트 보드"
④ 사람-AI 협업 KPI 직원의 AI 활용도, 성과 측정 "인간+AI 성과관리"
⑤ CEO AI (전문경영인 AI) 종합 판단, 오더 발행, 보고 생성 "AI 경영자"

Paperclip은 레이어 ①의 일부일 뿐이었다. 내가 만들려는 DECOHUB은 — 5개 레이어 전체였다.

Chapter 06

5개 레이어,
각각의 세계가 있었다

①
에이전트 오케스트레이션
"여러 AI를 하나의 팀으로 묶는 접착제"
Paperclip — Node.js+React, 조직도 모델링, 목표 계층, 예산 통제, 감사 로그 (MIT)
CrewAI — Python 기반 역할 모델, 20줄로 시작, FastAPI 호환
n8n — 422+ 앱 연동, 텔레그램 통합, 에이전트를 도구로 노출
Dify — RAG 지식 베이스, 에이전트 빌더 UI, API 배포
LangGraph — 상태 관리, 체크포인팅, 복잡한 의사결정 그래프
Paperclip CrewAI n8n Dify LangGraph
②
모니터링 & 옵저버빌리티
"각 에이전트가 살아있는지, 비용은 얼마인지"
Langfuse — LLM 호출 추적, 토큰 비용, 프롬프트 버전 관리 (오픈소스)
도구 호출 단위 추적 — 어떤 API를 언제 호출했는지
에이전트 헬스체크, 실행 트레이스, 행동 보고, 보안 이벤트
Langfuse LangSmith Composio
③
프로젝트·태스크 관리
"AI와 사람이 같은 태스크 보드에서 일하는 구조"
Linear 스타일 이슈 보드 UI — 프로젝트/상태 추적의 UX 표준
AI 에이전트와 사람 직원이 동일한 보드에서 할당받는 구조가 핵심
Linear DECOHUB 확장
④
사람-AI 협업 KPI
"직원이 AI를 얼마나 잘 쓰는지 측정"
AI 활용도 + 태스크 완료율 + 협업 품질 측정
이미 우리가 만든 평가체계(AI활용 40 + 역량 30 + 학습 30)를 확장
KPI Blueprint 기존 평가체계
⑤
CEO AI (전문경영인 AI)
"①~④의 모든 데이터를 종합 판단"
Paperclip CEO Agent — 미션→전략 목표 분해→직속 보고자 위임
OrchestrAI — 50~300개 에이전트 운영, 자동 전략 추천 기능
하나의 제품이 아니다. ①~④의 모든 데이터를 하나의 LLM에 넣어 종합 판단.
문실장을 이 레이어로 진화시키는 것이 가장 현실적인 경로.
문실장 진화 n8n + Claude API Paperclip CEO OrchestrAI

Paperclip 하나를 카피하는 것이 아니었다.
5개의 서로 다른 도메인이 결합된 시스템이고,
각 도메인마다 참조할 최적의 프로그램이 달랐다.

이 3개를 먼저 깔고, 나머지를 점진적으로 올리는 것이 가장 현실적이다.

Chapter 08

이건 과정이고
공부고, 스토리다

문실장 AI가 다운되고, 문실장 AI를 다시 세팅하고, 다시 가르치고, 자료를 보내고, 정리하고 — 그 많은 순간들을 지금 흘려보내고 있다.

하지만 안다. 지난 두 달 동안 느낀 것. 이건 과정이고, 공부고, 스토리다. 시간이 지나면 된다는 걸 알기 때문에 — 포기할 일은 없다.

각 에이전트가 진짜 뭘 하고 있는지 — 확인해야 한다. 문자만 띡 보내놓고 할루시네이션으로 "하고 있습니다"라고 하지는 않는지. CLAUDE.md에 넣어놨다. 그래도. 체크하고, 확인하고, 만들어 가야 한다.

제대로 만들어 볼 생각이다. 서버를 구축하고, 업무를 주고, 사람이 직접 이용하도록 만들 것이다.
